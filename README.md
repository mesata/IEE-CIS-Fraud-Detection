# IEE-CIS-Fraud-Detection
Machine Learning competition

## შეჯიბრის მიმოხილვა
IEEE-CIS Fraud Detection კონკურსის მიზანია ტრანზაქციების მონაცემების ანალიზი და იმის დადგენა, არის თუ არა კონკრეტული ტრანზაქცია თაღლითური (isFraud). მონაცემები მოიცავს ტრანზაქციულ ინფორმაციასა და მომხმარებელთა შესახებ სხვადასხვა დეტალს. ამოცანა ფასდება ROC AUC Score-ით, რაც გვიჩვენებს მოდელის უნარს, სწორად განასხვავოს თაღლითური და ლეგიტიმური ქმედებები. ჩემი სტრატეგია იყო, დამეწყო რაღაც მარტივი მოდელით, დავკვირვებულიყავი რა პრობლემები ჰქონდა და შემდგომი მოდელები ამ პრობლემების გადაჭრისუნარიანობით ამერჩია, სანამ ოპტიმალურ შედეგამდე არ მივიდოდი

## რეპოზიტორიის სტრუქტურა

## Feature Engineering
ლოჯისტიკურ რეგრესიაში მონაცემების მომზადება დაიწყო ორი ძირითადი ცხრილის Transaction და Identity გაერთიანებით, რამაც საშუალება მოგვცა მომხმარებლის ქცევა და მისი მოწყობილობის მახასიათებლები ერთ სივრცეში დაგვენახა. პირველი კრიტიკული ნაბიჯი იყო ინფორმაციული "ხმაურის" მოცილება. სრულად წაიშალა ის სვეტები, სადაც მონაცემების 80%-ზე მეტი არ არსებობდა (NaN). დარჩენილი ცარიელი მნიშვნელობების შესავსებად გამოვიყენე მედიანა, რადგან ის გაცილებით მდგრადია გადახრების მიმართ, ვიდრე საშუალო არითმეტიკული. კატეგორიული ცვლადები რიცხვით ფორმატში LabelEncoder-ით გადავიყვანე, ხოლო დროის დინამიკის დასაჭერად შევქმენი TransactionHour მახასიათებელი, რომელმაც აჩვენა, რომ თაღლითობის ალბათობა დღე-ღამის გარკვეულ მონაკვეთებში იცვლება.

## Feature Selection
ლოჯისტიკური რეგრესიისთვის, რომელიც პარამეტრულ მოდელს წარმოადგენს, განსაკუთრებით მნიშვნელოვანია მონაცემთა ხარისხი და გარკვეული სტატისტიკური დაშვებების დაცვა. ვინაიდან ლოჯისტიკური რეგრესია მგრძნობიარეა ზედმეტი და ურთიერთდაკავშირებული ინფორმაციის მიმართ, გადავწყვიტე ამეგო კორელაციის მატრიცა. <img width="454" height="409" alt="image" src="https://github.com/user-attachments/assets/a45b74e6-7aa4-4f2d-8427-4a4ab5927e6d" />
სურათზევე ჩანს, რომ c1  და c2 ცვლადებს შორის კორელაცია 1ის ტოლია, ანუ ერთსა და იმავე ინფორმაციას ატარებენ. გადავწყვიტე, რომ c2 წამეშალა. როდესაც ორი ცვლადი, მაგალითად C1 და C2, 1-თან მიახლოებულ კორელაციაშია, მოდელი ვეღარ ახერხებს თითოეული მათგანის ინდივიდუალური წონის სწორად განსაზღვრას. ეს იწვევს კოეფიციენტების არასტაბილურობას, სადაც მონაცემებში მცირე ცვლილებამაც კი შეიძლება მოდელის პასუხი რადიკალურად შეცვალოს.
