# IEE-CIS-Fraud-Detection
Machine Learning competition

## შეჯიბრის მიმოხილვა
IEEE-CIS Fraud Detection კონკურსის მიზანია ტრანზაქციების მონაცემების ანალიზი და იმის დადგენა, არის თუ არა კონკრეტული ტრანზაქცია თაღლითური (isFraud). მონაცემები მოიცავს ტრანზაქციულ ინფორმაციასა და მომხმარებელთა შესახებ სხვადასხვა დეტალს. ამოცანა ფასდება ROC AUC Score-ით, რაც გვიჩვენებს მოდელის უნარს, სწორად განასხვავოს თაღლითური და ლეგიტიმური ქმედებები. ჩემი სტრატეგია იყო, დამეწყო რაღაც მარტივი მოდელით, დავკვირვებულიყავი რა პრობლემები ჰქონდა და შემდგომი მოდელები ამ პრობლემების გადაჭრისუნარიანობით ამერჩია, სანამ ოპტიმალურ შედეგამდე არ მივიდოდი

## Feature Engineering
ლოჯისტიკურ რეგრესიაში მონაცემების მომზადება დაიწყო ორი ძირითადი ცხრილის Transaction და Identity გაერთიანებით, რამაც საშუალება მოგვცა მომხმარებლის ქცევა და მისი მოწყობილობის მახასიათებლები ერთ სივრცეში დაგვენახა. პირველი კრიტიკული ნაბიჯი იყო ინფორმაციული "ხმაურის" მოცილება. სრულად წაიშალა ის სვეტები, სადაც მონაცემების 80%-ზე მეტი არ არსებობდა (NaN). დარჩენილი ცარიელი მნიშვნელობების შესავსებად გამოვიყენე მედიანა, რადგან ის გაცილებით მდგრადია გადახრების მიმართ, ვიდრე საშუალო არითმეტიკული. კატეგორიული ცვლადები რიცხვით ფორმატში LabelEncoder-ით გადავიყვანე, ხოლო დროის დინამიკის დასაჭერად შევქმენი TransactionHour მახასიათებელი, რომელმაც აჩვენა, რომ თაღლითობის ალბათობა დღე-ღამის გარკვეულ მონაკვეთებში იცვლება.

## Feature Selection
ლოჯისტიკური რეგრესიისთვის, რომელიც პარამეტრულ მოდელს წარმოადგენს, განსაკუთრებით მნიშვნელოვანია მონაცემთა ხარისხი და გარკვეული სტატისტიკური დაშვებების დაცვა. ვინაიდან ლოჯისტიკური რეგრესია მგრძნობიარეა ზედმეტი და ურთიერთდაკავშირებული ინფორმაციის მიმართ, გადავწყვიტე ამეგო კორელაციის მატრიცა. <img width="454" height="409" alt="image" src="https://github.com/user-attachments/assets/a45b74e6-7aa4-4f2d-8427-4a4ab5927e6d" />

გადავწყვიტე, რომ წინა სამი ეტაპი (cleaning, Feature Engineering, Feature Selection) ყველა მოდელისთვის იგივენაირად შემესრულებინა, რადგან ყველაზე ოპტიმალურ შედეგებს იძლეოდა. 

## Training
### ლოჯისტიკური რეგრესია
მონაცემების მომზადების შემდეგ გადავედი მოდელირების ეტაპზე, სადაც პირველი ექსპერიმენტისთვის შევარჩიე ლოჯისტიკური რეგრესია. ჩემი მიზანი იყო შემექმნა სანდო Baseline, რომელიც მკაფიოდ მაჩვენებდა, რამდენად გამიჯვნადია მონაცემები წრფივ დონეზე. პროცესის ავტომატიზაციისა და მონაცემთა გაჟონვის თავიდან აცილების მიზნით გამოვიყენე Pipeline სტრუქტურა. მასში გავაერთიანე მონაცემთა შევსება, სტანდარტული სკალირება და უშუალოდ ჩემი მოდელი. სკალირება ამ შემთხვევაში კრიტიკულად მნიშვნელოვანი ნაბიჯი აღმოჩნდა, რადგან ლოჯისტიკური რეგრესია მგრძნობიარეა ცვლადების მასშტაბების მიმართ. მის გარეშე მოდელი ვერ შეძლებდა წონების სწორად დაბალანსებას და კონვერგენციას.
ვალიდაციის სტრატეგიად ავირჩიე მონაცემების გაყოფა ისე, რომ შენარჩუნებულიყო დროითი თანმიმდევრობა, რაც უმნიშვნელოვანესია ტრანზაქციების ანალიზისას. მოდელი უნდა სწავლობდეს წარსულზე და აკეთებდეს პროგნოზს მომავალზე. საბოლოო შედეგმა, 0.8042 AUC ქულამ, აჩვენა, რომ Baseline მოდელმა ნორმალურად იმუშავა, თუმცა შედეგების სიღრმისეულმა ანალიზმა სხვა სურათი გამოკვეთა.

ალბათობების განაწილების გრაფიკზე დაკვირვებისას გამოჩნდა, რომ მოდელი ძალიან დარწმუნებულია ლეგიტიმური ტრანზაქციების იდენტიფიცირებაში, თუმცა უჭირს თაღლითური ქმედებების გამოყოფა. გრაფიკზე არსებული Overlap ზონა მიუთითებს იმაზე, რომ წრფივი მოდელი ვერ აღიქვამს იმ კომპლექსურ პატერნებს, რომლებიც თაღლითობისთვისაა დამახასიათებელი. ეს არის კლასიკური Underfitting-ის მაგალითი, სადაც მოდელი ზედმეტად მარტივია მონაცემთა რეალური სირთულის აღსაწერად. ამ ამოცანაში Presicion - recall ურთიერთობა უმნიშვნელოვანესია. ლოჯისტიკურმა რეგრესიამ ეს ნაწილი კარგად ვერ დაიჭირა, ამიტომ უფრო რთულ მოდელებზე გადასვლა გადავწყვიტე.

ამ ექსპერიმენტის ფუნდამენტური მიზანი იყო იმ არაწრფივი და რთული კავშირების დაჭერა, რომლებიც თაღლითურ ტრანზაქციებს ახასიათებთ. IEEE-CIS მონაცემთა ბაზა შედგება მრავალი კატეგორიული და რაოდენობრივი ცვლადისგან, რომელთა შორის დამოკიდებულება ხშირად "პირობითია", მაგალითად, ტრანზაქციის გარკვეული მოცულობა შეიძლება იყოს საეჭვო მხოლოდ კონკრეტული ბარათის ტიპისა და დღის მონაკვეთის თანხვედრისას. სწორედ ასეთი პირობითი ლოგიკის მოდელირებისთვის გადაწყვეტილების ხე უკეთესი ინსტრუმენტი აღმოჩნდა.

ალგორითმის მუშაობის პრინციპი ეფუძნება მონაცემთა სივრცის რეკურსიულ დაყოფას. თითოეულ კვანძში მოდელი ირჩევს ისეთ ცვლადს და ისეთ "გამყოფ წერტილს", რომელიც მაქსიმალურად ზრდის ინფორმაციულ სისუფთავეს ან ამცირებს გაურკვევლობას. თაღლითობის აღმოჩენის კონტექსტში, ხემ ავტომატურად მოახდინა ცვლადების იერარქიზაცია. მაგალითად, მოდელმა პირველადი დაყოფისთვის გამოიყენა V-სვეტების გარკვეული ჯგუფები და C-ცვლადები, რაც მიუთითებს მათ გადამწყვეტ როლზე თაღლითური ქცევის იდენტიფიცირებაში.

თუმცა, გადაწყვეტილების ხის გამოყენებამ გამოკვეთა ამ ალგორითმის ერთ-ერთი მთავარი ტექნიკური გამოწვევა: მაღალი ვარიაცია. IEEE-CIS მონაცემთა მაღალი განზომილებიანობის პირობებში, ხე მიდრეკილია ზედმეტი დეტალიზაციისკენ. თუ ჩვენ არ შევზღუდავთ ხის სიღრმეს, მოდელი სწავლობს არა ზოგად კანონზომიერებებს, არამედ საწვრთნელ ბაზაში არსებულ შემთხვევით "ხმაურს", რაც იწვევს Overfitting-ს. ექსპერიმენტის ფარგლებში განხორციელდა ჰიპერპარამეტრების ფრთხილი ოპტიმიზაცია: სიღრმის პარამეტრების რეგულირებით შევძელი ბალანსის პოვნა მოდელის "დაზეპირების უნარსა" და მის გენერალიზაციის უნარს შორის.

მიუხედავად იმისა, რომ ცალკეულმა გადაწყვეტილების ხემ ვერ აჩვენა ისეთივე მაღალი AUC, როგორიც შემდგომმა ანსამბლურმა მოდელებმა, მან კრიტიკულად მნიშვნელოვანი როლი ითამაშა Feature Importance-ის (ცვლადების მნიშვნელობის) გაგებაში.

### Random Forest
მას შემდეგ, რაც გადაწყვეტილების ხემ აჩვენა მონაცემთა არაწრფივი სტრუქტურის დაჭერის უნარი, მაგრამ ამავე დროს გამოავლინა მაღალი მგრძნობელობა გადაჭარბებული დაზეპირების მიმართ, კვლევის შემდეგი ეტაპი დაეთმო Random Forest-ს. ეს მოდელი წარმოადგენს ანსამბლურ მიდგომას, რომელიც ეფუძნება "Bagging" (Bootstrap Aggregating) პრინციპს. მისი მთავარი იდეაა არა ერთი სრულყოფილი ხის შექმნა, არამედ მრავალი დამოუკიდებელი ხისგან შემდგარი "ტყის" აგება, სადაც საბოლოო გადაწყვეტილება მიიღება ხმათა უმრავლესობით. დიდ მონაცემებთან მუშაობისას Random Forest-მა ორი უმნიშვნელოვანესი უპირატესობა გამოავლინა. პირველი ეს არის Feature Randomness; თითოეული ხე იწვრთნება არა ყველა სვეტზე, არამედ მათ შემთხვევით ქვესიმრავლეზე. ამან საშუალება მოგვცა, რომ მოდელს არ მოეხდინა ფოკუსირება მხოლოდ რამდენიმე დომინანტურ ცვლადზე და აღმოეჩინა ფარული კავშირები ნაკლებად აშკარა სვეტებს შორისაც. მეორე ის, რომ მოდელი გახდა ბევრად უფრო სტაბილური "ხმაურის" მიმართ, რადგან ინდივიდუალური ხეების შეცდომები ანსამბლში ერთმანეთს აბალანსებს. ექსპერიმენტის ფარგლებში განსაკუთრებული ყურადღება დაეთმო n_estimators-ისა და max_features-ის ბალანსს. მიუხედავად იმისა, რომ ხეების რაოდენობის ზრდა აუმჯობესებდა AUC მაჩვენებელს, პარალელურად იზრდებოდა გამოთვლითი რესურსების ხარჯი. 

### AdaBoost
Random Forest-ის შემდეგ, სადაც აქცენტი ხეების დამოუკიდებელ, პარალელურ მუშაობაზე კეთდებოდა, კვლევა გადავიდა ბუსტინგის (Boosting) ალგორითმების შესწავლაზე, კერძოდ კი AdaBoost (Adaptive Boosting) მოდელზე. მიუხედავად იმისა, რომ ორივე მეთოდი გადაწყვეტილების ხეებს იყენებს, AdaBoost ფუნდამენტურად განსხვავებული ფილოსოფიით მოქმედებს. თუ Random Forest-ში ხეები ერთდროულად იზრდებიან და მათი შეცდომები საშუალოვდება, AdaBoost იყენებს მიმდევრობით (Sequential) მიდგომას.

AdaBoost-ის მუშაობის პრინციპი ეფუძნება "სუსტი შემსწავლელების" (Weak Learners) — ხშირად მხოლოდ ერთი დონის ხეების (Decision Stumps) — გაერთიანებას ძლიერ პროგნოზულ სისტემაში. მისი "ადაპტურობა" გამოიხატება იმაში, რომ ყოველი მომდევნო ხე ცდილობს გამოასწოროს წინა ხის მიერ დაშვებული შეცდომები. მოდელი მეტ "წონას" და ყურადღებას ანიჭებს იმ ტრანზაქციებს, რომლებიც წინა ეტაპზე არასწორად იქნა კლასიფიცირებული. IEEE-CIS მონაცემთა ბაზაში, სადაც თაღლითური ოპერაციები იშვიათია და რთულად დასაჭერი, ეს მიდგომა თეორიულად ძალიან ეფექტურია, რადგან მოდელი "იძულებულია" ისწავლოს ყველაზე რთული შემთხვევები.

თუმცა, ექსპერიმენტმა აჩვენა, რომ AdaBoost-ს გააჩნია გარკვეული სუსტი მხარეები ამ კონკრეტულ ამოცანაში. ის განსაკუთრებით მგრძნობიარეა მონაცემთა "ხმაურის" (Outliers) მიმართ, რადგან ცდილობს ნებისმიერ ფასად მოახდინოს არასწორი შემთხვევების კორექტირება. Random Forest-თან შედარებით, AdaBoost უფრო სწრაფად მივიდა ოპტიმიზაციის ზღვართან, თუმცა მისმა გამოყენებამ მოგვცა უმნიშვნელოვანესი ცოდნა ბუსტინგის მექანიზმებზე. სწორედ AdaBoost-ზე მიღებულმა გამოცდილებამ მიგვიყვანა საბოლოო, ყველაზე დახვეწილ ეტაპამდე — XGBoost-მდე, რომელმაც ბუსტინგის იდეა გრადიენტული ოპტიმიზაციის მეშვეობით კიდევ უფრო მაღალ დონეზე აიყვანა.
